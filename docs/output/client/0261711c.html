Hello, John A test page to demonstrate some basic capabilities.

Since yesterday:

  • Hand-created client and server entry points in route directories are gone. This logic is now generated at build-time.
  • I've created the basis for a powerful theme system that will be super simple to use. Install an npm package, change a line in your config, and recompile will reskin any site generated with snapsite.
  • This little demo uses several basic widgets implemented by the default theme.
    • breadcrumbs at the top of the page
    • page title
    • sitemap / copyright footer
    • In-site link helpers that abstract the details of URL's (can get tricky). For example: snapsite links back to the top of the site using a hash signature I don't know off the top of my head. In this case I use the route '/'
  • This little demo is a raw copy of the generated client app to github pages (free hosting).
